## Memory手法のトレンドと課題（2026年1月）

### 手法の根幹とトレンド
2026年1月の論文群から、メモリは受動的な情報の格納庫(RAG)から、エージェントの能動的な脳機能(Agentic Memory)へと完全に脱皮した。大きなトレンドは3点に集約される。第一に、メモリ操作自体をエージェントの行動空間として定義し、自律的に管理・操作する手法の一般化。第二に、単純なベクトル検索から、論理マップや木構造、グラフを用いた構造的なイベント管理への移行。第三に、過去のエピソードメモリを強化学習のソースとして活用し、経験から即座にスキルを抽象化する「経験学習」の深化である。

### 課題傾向
主な課題は、膨大な履歴の中での情報の「忘却と圧縮」の制御、および過去の記憶への固執（アンカリング）と未知の状況への適応（イノベーション）の対立解消である。また、単なる事実の想起ではなく、時間的・因果的な一貫性を保った「物語的な記憶再生」の実現や、認知科学的な妥当性、計算コストの劇的な削減と精度の両立が重要な論点となっている。

### 論文紹介

[Agentic Memory](https://arxiv.org/abs/2601.01885)

長期・短期メモリの統合管理をエージェント自らが学習する手法。従来の固定ルールに頼らず、タスクの文脈に応じてエージェントが「いつ書き込み、要約し、削除するか」を動的に選択する。メモリ操作をエージェントの行動の一部として定義し、推論効率を自律的に最適化する点が特徴。

[SimpleMem](https://arxiv.org/abs/2601.02553)

意味論的な非可逆圧縮に基づく効率的な生涯メモリフレームワーク。情報の密度を高めるための3段階パイプライン（構造化圧縮、オンライン統合、意図認識型検索）を提案。全履歴を保持する手法と比較して、精度を維持しつつトークン消費を最大30倍削減する圧倒的な効率性を実現。

[MEMRL](https://arxiv.org/abs/2601.03192)

エピソードメモリを強化学習のデータソースとして活用し、実行中にエージェントを自己進化させるフレームワーク。過去の成功・失敗経験を即座に次の行動へ反映するオンライン学習機能を備える。情報の保存ではなく、記憶を活用したリアルタイムな行動改善と未知環境への適応に特化。

[Memory Matters More](https://www.arxiv.org/abs/2601.04726)

イベントを核としたメモリ構造を論理マップとして構築。従来のRAGのような部分的な情報取得ではなく、イベント間の因果関係や時間的前後関係を保持することで、複雑な検索や深い推論を必要とするタスクで高い精度を発揮する。情報の点ではなく、論理的な線による推論の一貫性を重視。

[Controllable Memory Usage](https://arxiv.org/abs/2601.05107)

メモリへの依存度をユーザーが動的に制御できるフレームワーク。過去の記憶に固執するアンカリングと、新しい状況へのイノベーションのバランスを調整可能にする。メモリを静的なDBではなく、ユーザーの意図に合わせて振る舞いを変える調整可能なバイアスとして定義する。

[Inside Out: Core Memory Trees](https://arxiv.org/abs/2601.05171)

木構造を用いて長期的なユーザープロファイルを管理する手法。強化学習で訓練された軽量な管理ユニットが、木の更新や枝刈りを自律的に判断する。平坦なテキスト連結と異なり、構造化された木を採用することで、パーソナリティの一貫性維持と検索精度の劣化防止を両立。

[MineNPC-Task](https://arxiv.org/abs/2601.05215)

Minecraft内のエージェントのための新しい評価スイート。エキスパートプレイヤーの対話から得られたタスクを構造化。記憶の読み書き、計画、事前条件チェックなどのイベントを詳細に捕捉し、マルチステップな実環境タスクにおけるメモリ管理能力を厳密に計測するためのベンチマーク。

[PACEvolve](https://arxiv.org/abs/2601.10657v1)

進捗認識型(Progress-Aware)の進化フレームワーク。階層的なコンテキスト管理により、過去の実験履歴による「コンテキスト汚染」を防ぎ、バックトラッキング手法で探索を効率化。メモリを実験の管理ログとして扱い、探索と活用のバランスを高度に最適化。

[The AI Hippocampus](https://arxiv.org/abs/2601.09113v1)

LLMのメモリを人間の海馬の機能に照らして整理したサーベイ。暗黙的・明示的・エージェント的メモリの3層タクソノミーを提唱。技術提案よりも認知科学的視点を重視し、現状の評価指標が人間の心理の一致度を測るのに不十分であることを指摘。

[MemoBrain](https://arxiv.org/abs/2601.08079v1)

メモリを推論の核となる「エージェントの脳」の実行機能の一部として再定義。単なる外部検索システムではなく、意思決定プロセスと密に結合した作業領域としてのメモリを提案。情報の格納場所から、より能動的な推論用実行ユニットへの進化を目指す。

[AtomMem](https://arxiv.org/abs/2601.08323v1)

メモリ操作を最小単位の演算（アトミック操作）に分解し、それらを学習可能にする手法。標準化されたインターフェースを提供することで、異なるモデル間でのメモリ共有や、複雑なワークフロー内でのメモリ操作ের再利用性を高める。アーキテクチャのモジュール化が核心。

[Fine-Mem](https://arxiv.org/abs/2601.08435v1)

メモリ管理の最適化に細粒度フィードバックを導入。最終的なタスク成否だけでなく、ステップごとのメモリ操作に即時報酬を与える。強化学習を用いたメモリ最適化において課題となる報酬の疎性を解決し、どの記憶操作が有効だったかのクレジット割り当てを正確に行う。

[Structured Episodic Event Memory](https://arxiv.org/abs/2601.06411v1)

事実のグラフメモリと物語的なエピソードメモリを統合した階層フレームワーク。情報の断片をエピソードイベントフレームとして構造化する。単なる情報の抽出ではなく、認知フレーム理論に基づいた一貫したストーリーとしての記憶再生を狙う。

[Active Context Compression](https://arxiv.org/abs/2601.07190v1)

粘菌の探索戦略に想を得た自律的圧縮手法。外部の受動的要約ではなく、エージェント自身がいつ重要な知見を抽出し、不要な履歴を剪定するかを決定する。ノイズを排除してトークン消費を削減し、ソフトウェア開発などの長文コンテキスト下での推論低下を防ぐ。

[From Storage to Experience](https://www.preprints.org/manuscript/202601.0618)

メモリが静的な格納庫から動的な経験獲得へと進化したトレンドを分析したサーベイ。初期のプロンプト拡張から現在の経験学習へのシフトを定義。構造化スキーマや手続き的プリミティブによる知識の再利用を次世代の鍵と位置づける。

[AutoRefine](https://arxiv.org/abs/2601.22758)

タスクの実行軌跡を分析し、再利用可能な専門知識へと自動的に変換・洗練する仕組み。一度の経験を抽象化してメモリに保存し、将来の類似課題に直接適用する。単なる情報の再利用を超え、経験を汎用的なスキルへと昇華させる継続的自己改善の手法。
